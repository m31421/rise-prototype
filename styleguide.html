6. Semantic headings (production)

Use plain h1–h5 in markup. Base styles use font-size: var(--font-size-h*). Section-specific size differences are done by setting --font-size-h2 (or another level) on a layout ancestor—custom properties inherit, so h2 picks up the value without extra font-size rules. Avoid BEM-style heading classes and avoid new --font-size-{section}-* tokens.

Level tokens (:root)

--font-size-h1 → var(--font-size-display) clamp(3rem, 5.5vw + 1.75rem, 9rem)
--font-size-h2 → var(--font-size-section-title) clamp(36px, 8vw, 64px)
--font-size-h3 → var(--font-size-26) 26px
--font-size-h4 → var(--font-size-16) 16px
--font-size-h5 → var(--font-size-14) 14px

Utilities (accent lines inside headings)

.text-primary — color: var(--color-text-primary)
.text-brand — color: var(--color-brand-orange)
.heading-line — display: block (stack lines; .nmc h2 adds spacing between adjacent lines)

Where layouts override headings

These selectors in global.css adjust size, color, or treatment for a specific block. Headings in HTML stay unclassed.

Selector Purpose
.hero h1 Gradient fill (size from base h1 / --font-size-h1)
.ioyt Sets --font-size-h2 to a larger clamp for the IOYT headline
.ioyt__text-layer h2 Font family, color, flex layout (inherits h2 size from .ioyt)
.nmc Sets --font-size-h2: var(--font-size-section-title-sm)
.nmc h2 Letter-spacing and line-height (inherits size)

.gofaster__team-col h3 Margin below the heading in the team column
